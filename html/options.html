#2.1.30
- Парсинг JSON Patch: Логика parsePage переведена на обработку формата JSON Patch (window.STATE_PATCHES), что вернуло сборку данных страниц после обновления Яндекса.
- Сборка дерева данных: Внедрен метод фильтрации с развертыванием через .flat() и сборкой путей по слэшам в объект finalTree, заменивший строки со слэшами на красивую вложенную JS-структуру.
- Перехват скрытого аудио: Внедрена подмена глобального метода document.createElement('audio'), позволившая ловить ссылки на аудиопотоки напрямую из динамической памяти в обход DOM-дерева.
- Новый парсер Track ID: Регулярное выражение переписано под новый формат MP4/AAC-стриминга (strm.yandex.net), обеспечив точное извлечение ID трека из четвертого числового сегмента ссылки.
- Красивая FAB-кнопка: Внедрена круглая зеленая кнопка скачивания текущей песни в углу экрана с визуальной сменой статусов («Клубника», «MP3...») во время работы.
- Автономное скачивание: Логика кнопки переведена на прямую генерацию ссылок через fetchFileInfoOne, что исключило отправку sendMessage и убрало ошибку связи при обновлении расширения.
- Безопасный интерцептор: Исключен риск зависания SPA-интерфейса за счет изоляции контекста (this) через стрелочные функции в перехватчиках сетевых запросов.
- Очистка кэша сессии: Добавлено полное принудительное удаление старого ключа appYa_page из localStorage перед сохранением нового состояния, исключив накопление мусора от старых сессий.
