Единовременная закачака за один проход

Переменные:
Например: music/%dir%/%genre%/%year%/%artist%/%album% %dir% - Директория откуда качать %genre% - Жанр %year% - Год %artist% - Артист %album% - Альбом
Если указать music без переменных, закачка идет стандартно согласно текущему плейлисту, артисту

music_name.mp3 





#2.1.30
    - Парсинг JSON Patch: Логика parsePage переведена на обработку формата JSON Patch (window.STATE_PATCHES), что вернуло сборку данных страниц после обновления Яндекса.
    - Сборка дерева данных: Внедрен метод фильтрации с развертыванием через .flat() и сборкой путей по слэшам в объект finalTree, заменивший строки со слэшами на красивую вложенную JS-структуру.
    - Перехват скрытого аудио: Внедрена подмена глобального метода document.createElement('audio'), позволившая ловить ссылки на аудиопотоки напрямую из динамической памяти в обход DOM-дерева.
    - Новый парсер Track ID: Регулярное выражение переписано под новый формат MP4/AAC-стриминга (strm.yandex.net), обеспечив точное извлечение ID трека из четвертого числового сегмента ссылки.
    - Красивая FAB-кнопка: Внедрена круглая зеленая кнопка скачивания текущей песни в углу экрана с визуальной сменой статусов («Клубника», «MP3...») во время работы.
    - Автономное скачивание: Логика кнопки переведена на прямую генерацию ссылок через fetchFileInfoOne, что исключило отправку sendMessage и убрало ошибку связи при обновлении расширения.
    - Безопасный интерцептор: Исключен риск зависания SPA-интерфейса за счет изоляции контекста (this) через стрелочные функции в перехватчиках сетевых запросов.
    - Очистка кэша сессии: Добавлено полное принудительное удаление старого ключа appYa_page из localStorage перед сохранением нового состояния, исключив накопление мусора от старых сессий.
        

#2.1.27
    - Исправлен клон ответа: Вызов response.clone() вынесен из setTimeout вверх до return, что убрало ошибку Response body is already used.
    - Убрано зацикливание: Добавлен пропуск проверки для запросов с byVectorserver, остановивший бесконечные повторные вызовы.
    - Фильтр дубликатов: Внедрен предохранитель this.lastTrackId, блокирующий одновременные запросы плеера для одной песни.
    - Переход на Linux API: Запросы переведены на мобильный эндпоинт с авторизацией по OAuth, что убрало ошибки 403 и not-allowed.
    - Исправление подписи: Функция generateSign переписана на правильный алгоритм MD5 Hex вместо неверного HMAC SHA-256 Base64.
    - Обход CORS: Скачивание переведено в режим redirect: 'manual' для безопасного извлечения прямой ссылки на аудиофайл.

#2.1.25
    - Перехват POST-запросов: Добавлена поддержка извлечения данных из тела (body) запросов, что позволило фиксировать trackId при отправке статистики в /plays.
    - Унификация Fetch: Метод мониторинга теперь корректно обрабатывает запросы, созданные как через объект Request, так и через прямую передачу URL/Options.
    - Безопасное клонирование: Реализовано клонирование потоков запроса и ответа (clone()), предотвращающее ошибки "body already used" в оригинальном плеере.
    - Приоритизация trackId: Улучшена логика определения трека — теперь скрипт ищет ID сначала в теле запроса, и только потом в URL-параметрах.
    - Оптимизация производительности: Парсинг JSON из ответов сервера теперь происходит только при наличии валидного application/json в заголовках.
